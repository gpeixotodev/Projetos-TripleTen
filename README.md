Projetos Práticos de QA – TripleTen

- Planejamento e execução de testes manuais para aplicações Web, Mobile e APIs REST.
- Criação de casos de teste utilizando Classes de Equivalência e Análise de Valores Limite.
- Registro e acompanhamento de bugs no Jira.
- Testes de API utilizando Postman, validação de códigos HTTP e payloads JSON.
- Consultas SQL para validação de dados em banco de dados.
- Manipulação de logs e arquivos em ambiente Linux (Bash/Cygwin).
- Desenvolvimento de testes automatizados em Python, utilizando Selenium WebDriver, Pytest e o padrão Page Object Model (POM).
- Controle de versão com Git.

Descrição detalhada de cada sprint:
- Sprint 1 – Testes Manuais e Relatório de Bugs (Urban Routes)

Descrição:
Elaboração de casos de teste funcionais para aplicação web, execução de testes exploratórios e funcionais, identificação e documentação de bugs utilizando Jira.

Tecnologias/Ferramentas:
* Testes Manuais
* Jira
* Google Sheets
* Test Design

- Sprint 2 – Design de Testes

Descrição:
Criação de classes de equivalência, análise de valores limite e desenvolvimento de casos de teste para validação de formulários e regras de negócio.

Tecnologias/Ferramentas:
* Test Design
* Classes de Equivalência
* Análise de Valores Limite
* Google Sheets

- Sprint 3 – Linux, SQL e Jira

Descrição:
Análise de logs utilizando comandos Linux, consultas SQL para validação de dados em banco de dados relacional e registro de defeitos no Jira.

Tecnologias/Ferramentas:
* SQL
* PostgreSQL
* Linux (Bash/Cygwin)
* Git Bash/Cygwin
* Jira

- Sprint 4 – Testes de API REST

Descrição:
Planejamento e execução de testes em APIs REST, criação de casos de teste para endpoints, validação de respostas HTTP, identificação de falhas e documentação de bugs.

Tecnologias/Ferramentas:
* Postman
* API REST
* JSON
* HTTP
* Jira

- Sprint 5 – Testes Mobile

Descrição:
Execução de testes funcionais em aplicação mobile utilizando checklist de validação, registro de inconsistências e documentação de defeitos.

Tecnologias/Ferramentas:
* Testes Mobile
* Android
* Jira
* Google Sheets

- Sprint 6 – Automação de Testes Web

Descrição:
Desenvolvimento de testes automatizados para aplicação web utilizando Selenium WebDriver, aplicando boas práticas de automação e execução com Pytest.

Tecnologias/Ferramentas:
* Python
* Selenium WebDriver
* Pytest
* POM (Page Object Model)

- Sprint 7 – Automação de API

Descrição:
Desenvolvimento de testes automatizados para APIs REST, validação de respostas, códigos de status e regras de negócio utilizando Python.

Tecnologias/Ferramentas:
* Python
* Pytest
* Requests
* JSON
* API REST

- Sprint 8 – Projeto Final de QA

Descrição:
Projeto completo de Quality Assurance envolvendo planejamento e execução de testes manuais para aplicação web e APIs. Desenvolvimento de casos de teste baseados em classes de equivalência e análise de valores limite, validação de regras de negócio, documentação de defeitos e acompanhamento dos resultados dos testes. Os cenários abrangeram funcionalidades web, dispositivos móveis e endpoints REST. Os arquivos do projeto mostram casos de teste para Web, API e validações funcionais estruturadas.

Tecnologias/Ferramentas:
* Python
* Selenium WebDriver
* Pytest
* Postman
* API REST
* JSON
* Jira
* SQL
Linux (Cygwin/Bash)
Git
